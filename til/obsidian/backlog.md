# Obsidian 학습 백로그

## 선행 지식
- [ ] [[YAML Frontmatter]] - 노트 상단에 `---`로 감싸는 메타데이터 블록 형식
- [ ] [[PKM]] - 개인 지식 관리(Personal Knowledge Management)의 기본 개념과 목적

## 핵심 개념
- [ ] [[Vault]] - Obsidian의 기본 저장 단위. 로컬 파일시스템 기반 폴더 구조
- [ ] [[Wikilink와 Backlink]] - `[[이중 대괄호]]`로 노트 간 양방향 연결을 만드는 핵심 메커니즘
- [ ] [[Properties]] - YAML 기반 노트 메타데이터 시스템. 타입 지정, 검색/필터에 활용
- [ ] [[Graph View]] - 노트 간 연결을 시각적 그래프로 보여주는 기능
- [ ] [[Core Plugins]] - 내장 플러그인 (Command Palette, Daily Notes, Templates, Bookmarks 등)
- [ ] [[검색과 태그]] - 노트 검색, `#태그` 시스템, 필터링 방법
- [ ] [[Callout과 Embed]] - `> [!callout]` 블록과 `![[노트]]` 임베딩 문법
- [ ] [[Map of Content]] - 노트들을 체계적으로 조직하는 허브 노트(MOC) 패턴
- [ ] [[Zettelkasten]] - 원자적 노트와 연결 기반 지식 관리 방법론

## 심화
- [ ] [[Community Plugins]] - 커뮤니티 플러그인 생태계 (Dataview, Templater, Calendar 등)
- [ ] [[Dataview]] - SQL 유사 쿼리로 노트를 데이터처럼 조회하는 플러그인
- [ ] [[Templater]] - 동적 변수와 JavaScript를 지원하는 고급 템플릿 플러그인
- [ ] [[Obsidian Git]] - Obsidian 내에서 Git 자동 커밋/동기화. GitHub 연동과 버전 관리
- [ ] [[Canvas]] - 무한 화이트보드. 노트, 이미지, 카드를 자유 배치하는 시각적 도구
- [ ] [[Bases]] - 노트를 데이터베이스 뷰로 편집/정렬/필터링하는 기능 (2025년 출시)
- [ ] [[Obsidian Sync와 Publish]] - E2E 암호화 동기화 서비스와 웹 퍼블리싱 기능
- [ ] [[Plugin 개발]] - TypeScript API, Plugin 클래스 라이프사이클, esbuild 빌드 시스템
- [ ] [[CSS Snippets과 테마]] - CSS를 통한 외관 커스터마이징과 커뮤니티 테마 적용
- [ ] [[Hotkeys 커스터마이징]] - 단축키 설정과 hotkeys.json 직접 편집으로 워크플로우 최적화
